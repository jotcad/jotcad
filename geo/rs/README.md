# Ruled Surface (RS) Library

This library provides high-performance algorithms for generating optimal **Ruled Surfaces** (triangulated ribbons or tubes) between two or more 3D polygonal chains. It is integrated into the JotCAD Mesh-VFS via the `jot/Rule` operator.

## 1. Core Concept
A ruled surface is generated by connecting points on two boundaries with straight lines (rulings). In a discrete mesh, this is achieved by finding the optimal triangulation of the quad-strip formed between the chains.

## 2. Algorithms & Strategies

### Triangulation Search
- **Dijkstra (Shortest Path):** The primary engine for finding the mathematically optimal triangulation between two open chains of size $M$ and $N$. It constructs a directed acyclic graph (DAG) of all possible triangles and finds the minimum-cost path from $(0,0)$ to $(M,N)$. 
  - **Complexity:** $O(MN)$
- **Yen's Algorithm:** Used when $K > 1$ results are requested. It generates the next-best triangulations by branching off the shortest path.
- **LinearAll (Exhaustive):** A permutation-based search used for small polylines where topology is highly constrained.

### Seam Alignment (Loop-to-Loop)
When ruling closed loops, the "start" point of the triangulation is arbitrary. The library must find the optimal "twist" (shift) between the loops.
- **Exhaustive Seam Search:** Iterates through all $N$ possible vertex shifts, performing a full triangulation for each. This is optimal but $O(N^3)$ total. 
- **Simulated Annealing (SA):** A stochastic search used for high-resolution loops where $O(N^3)$ is too slow. It samples the shift space to find a near-optimal alignment.

## 3. Optimization Objectives
The library can minimize different geometric properties by swapping the `Objective` template:
- `MinArea`: Minimizes the total surface area (default).
- `MinBendingEnergy`: Minimizes the change in surface normals between adjacent triangles.
- `MaxConvexity`: Favors triangulations that stay close to the convex hull of the profiles.
- `MinMeanCurvatureVariation`: Minimizes the variation in mean curvature across the surface.

## 4. Topological Handling
- **Open Polylines:** Triangulated as a flat ribbon.
- **Closed Loops:** Automatically "stitched" at the seam to form a continuous tube (manifold surface).
- **Holes:** The library uses a **Linear Assignment Solver** to pair multiple loops between two shapes (e.g., matching the outer boundary of Shape A to Shape B, and the hole of Shape A to the hole of Shape B).

## 5. Performance Notes (Future Maintenance)
- **Fast Path:** For $K=1$ (the standard CAD case), the library uses a direct Dijkstra implementation that bypasses expensive graph copies.
- **Vertex Count:** Complexity scales $O(N^3)$ for loops. For $N > 200$, consider implementing a centroid-based heuristic alignment to drop to $O(N^2)$.
- **Exact vs. Inexact:** While the JotCAD kernel uses exact predicates, this library casts to `double` for the search phase to maintain high throughput, then re-integrates into the exact VFS.

## 6. Directory Map
- `ruled_surfaces_base.h`: Foundational types and Mesh/Soup conversion.

- `ruled_surfaces_join_strategy_naive.h`: Combinatorial hole matching.
- `objective.h`: Interface for optimization objectives.
